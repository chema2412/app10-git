Hyper Text Markup Language ("HTML")

Paginas Estáticas

HTML Dio inicio en el año de 1993, Desglosando eso se hizo una version de de html nueva HTML 2.0 surgiendo en el año de 1995. A inicios del año de 1997 se dio a conocer una version nueva de HTML ("HTML 3.2"). Llegando asi hasta las paginas dinamicas. Las Paginas WEB Estaticas Son Basicamente informativas y estan enfocadas principalmente a mostrar una información permanente donde el navegante se limita a obtener dicha información sin poder intercactuar con la pagina visitada

Paginas Dinamicas

Estas Se dieron a conocer en el año de 1998 ("HTML 4.0") y 1999 ("HTML 4.01"). Las Paginas Web Dinamicas son aquellas en que la información presentada se genera a partir de una petición del usuario de la pagina, Esto se Hace posible porque una pagina dinamica viene asociada con una base de datos desde la que permite visualizar la información contenida en ella

Aplicaciones WEB

Estas Aplicaciones revolucionan el lenguaje de HTML haciendo una version mejorada y muy facil de utlizar "HTML 5" se dio a conocer en el año de 2014. Las aplicaciones web reciben este nombre porque se ejecutan en la internet. Es decir que los datos o los archivos en los que trabajas son procesados y almacenados dentro de la web. Estas aplicaciones, por lo general, no necesitan ser instaladas en tu computador. El concepto de aplicaciones web está relacionado con el almacenamiento en la nube. Toda la información se guarda de forma permanente en grandes servidores de internet y nos envian a nuestros dispositivos o equipos los datos que requerimos en ese momento, quedando una copia temporal dentro de nuestro equipo.

HTML

Etiquetas

El código en HTML está formado por un conjunto de etiquetas (tags). El navegador interpreta estas etiquetas al momento de cargar la página. La mayoría de las etiquetas vienen en pares, es decir: ("Etiqueta Apertura = Inicio") ("Etiqueta Cierre = Final (contienen una diagonal /)")

Tenemos algunas otras etiquetas que no vienen en pares, es decir, solo tiene etiqueta de apertura por ejemplo: ("Img" , "input" , "br" ).

Elementos HTML

Las etiquetas junto con su contenido, forman lo que conocemos como un elemento HTML. ("abre 2h1") h1> Ejemplo h1> ("Cierra "h1")

Atributos En las Etiquetas

La etiqueta de apertura de un elemento puede incluir varios atributos separados por espacios. Los atributos deben aparecer solo en las etiquetas de apertura. Algunos elementos requieren ciertos atributos de forma obligatoria, mientras que otros son opcionales. La etiqueta de apertura de un elemento puede contener atributos que proporcionan más información sobre el elemento. (abre "p" ) p class=“mensaje”> Hola Mundo /p> (Cierra "p").La etiqueta de apertura de un elemento puede incluir varios atributos separados por espacios. Los atributos deben aparecer solo en las etiquetas de apertura. Algunos elementos requieren ciertos atributos de forma obligatoria, mientras que otros son opcionales.

Elementos De HTML

Todos los elementos HTML se pueden dividir en dos tipos básicos: ("Elementos en bloque","Elementos en línea").Los elementos pueden anidarse, es decir, podemos tener elementos HTML dentro de otro(s) elementos.

Elementos en Bloque

Un elemento en bloque es aquel que contiene un bloque significativo de contenido que debería mostrarse en su propia línea. Por ejemplo: párrafos, tablas, listas, etc…

Elementos En Linea

Los elementos en línea usualmente contienen menos texto y se muestra junto a otro texto en la misma línea, cómo cuando se enfatizan palabras en una oración

Atributos Estandar HTML

Class

Indica la clase o clases a las cuales un elemento pertenece. Cualquier número de elementos pueden pertenecer a una clase. Un solo elemento puede pertenecer a muchas clases

Style

Especifica las propiedades CSS de un elemento

ID

Especifica un identificador único para un elemento

Formularios

Un formulario permite a un usuario ingresar datos. La información que se ingresa se manda a un servidor en un grupo de datos que consisten del nombre de los elementos y sus valores.Para hacer uso de un formulario, utilizaremos las etiquetas: ("form> /form>") Requiere de los atributos action y method, los cuales son la URL del form-handler y el método HTTP por el cual se envían los datos, respectivamente.El valor por default de action es la URL del documento actual.El valor por default de method es get (el otro valor válido es post)form method=“get” action=“url”>/form>.input> Muchos de los elementos de un formulario se crean con este elemento. Sus 3 atributos más comunes son: ("name","type","value")

Tipos De Validación de Formularios

Valor Faltante

Este tipo de validación se ocasiona por el nuevo atributo “required” el cual provoca que si un campo está vacío, se ocasione un error, por lo tanto será inválido.

Diferente Tipo

Este tipo de validación ocurre por ejemplo cuando tenemos un "input type=”email”>" Si estamos en un navegador que soporte este atributo, entonces el navegador sabrá que un email debe de llevar un símbolo @, si no lo encuentra provocará una invalidación.

por Patrón

Este tipo de validación ocurre cuando nosotros definimos un patrón a seguir dentro de nuestro input, por ejemplo:, ("input type=”text” pattern=”[0-9]{16}”>"). Este patrón indica que debe de haber 16 caracteres los cuales podrán ser dígitos del0 al 9, caso contrario será inválido.

CSS

"Cascading Style Sheets u "Hojas de Estilo en Cascada"

es un lenguaje usado para definir y crear la presentación de un documento estructurado escrito en HTML. CSS separa la estructura de un documento de su presentación. La información de estilo puede ser definida en un documento separado o en el mismo documento HTML.

Evolución De CSS

El organismo W3C (World Wide Web Consortium), encargado de crear todos los estándares relacionados con la web, propuso la creación de un lenguaje de hojas de estilos específico para el lenguaje HTML 2 Propuestas: "CHS (Cascading HTML Sheets) por Hakon Wium Lie". "SSP (Stream-based Style Sheet Proposal) por Bert Bos".

Como Agregar Hojas De Estilo

Existen tres formas de instertar CSS. Un estilo externo (“nombre_archivo.css ”). Un estilo interno ("style> h1 {color: red;} /style>"). Un estilo en la linea html (inline) ("p style="color:red; margin-left:20px">Un parrafo/p>")

Sintaxis De CSS

CSS tiene una sintaxis sencilla, que usa palabras clave (inglés) para especificar los nombres de las propiedades y la mayoría de los valores. Se especifica el elemento o elementos a aplicar el estilo (se pueden usar etiquetas, id o clases separadas o se pueden apilar).Vienen en pares de nombre/valor: name:value;Cada línea terminan con ";"

CSS

Tipos De Selectores

Definen que elementos HTML serán afectados por las declaraciones. Existen varios tipos de selectores, "Elementos Clases", "ID’s", "Descendientes", "Hijos Adyacentes (hermanos o siblings)", etc.

Selector Universal

El selector universal, selecciona todos los elementos de un documento HTML.Se denota por un “*”, * { margin:0;, padding:0;, box-sizing: border-box; }

Selectores tipo Etiqueta

Selectores tipo etiqueta o de elementos, son los más simples y comunes. Ejemplo h1 {color:blue;}body {background-color:red;}

Selectores de Clase

Se utiliza para especificar un estilo a un grupo de elementos con la misma clase.Su principal característica, es que se identifica por un punto. “.”

Selectores de ID

Se utiliza para especificar un estilo a un elemento mediante su id. Su principal característica es que se identifica por un sharp o gato “#”.("div id=”contenido”> Esto es un div /div>"), #contenido { background-image: url(“perro.jpg”); }

Selectores Descendientes

Afectan a elementos que están contenidos dentro de otro(s) elementos. Lo usamos mediante el uso de un espacio entre cada elemento. "ul li {font-size: 17px;margin-top: 14px;}"

Agrupación de Selectores

La agrupación de selectores se utiliza cuando uno o más selectores tienen las mismas propiedades.Esto nos ayudará a ahorrar líneas de código. Agrupamos con el uso de la coma “,”

PHP

Introducción

PHP (acrónimo recursivo de PHP: Hypertext Preprocessor) es un lenguaje de código abierto muy popular especialmente adecuado para el desarrollo web y que puede ser incrustado en HTML.

Lo mejor de utilizar PHP es su extrema simplicidad para el principiante, pero a su vez ofrece muchas características avanzadas para los programadores profesionales. No sienta miedo de leer la larga lista de características de PHP. En unas pocas horas podrá empezar a escribir sus primeros scripts.

Variables

PHP admite el concepto de funciones variables. Esto significa que si un nombre de variable tiene paréntesis anexos a él, PHP buscará una función con el mismo nombre que lo evaluado por la variable, e intentará ejecutarla. Entre otras cosas, esto se puede usar para implementar llamadas de retorno, tablas de funciones, y así sucesivamente. Las funciones variables no funcionarán con constructores de lenguaje como echo, print, unset(), isset(), empty(), include, require y similares. Utilice funciones de envoltura para hacer uso de cualquiera de estos constructores como funciones variables.
